# AWS Important Concepts (Quick Revision Notes)

## Core Services

### EC2 (Elastic Compute Cloud)
- Provides virtual servers in the cloud
- Scalable compute capacity
- Pay only for what you use

### S3 (Simple Storage Service)
- Object storage for files and backups
- High durability and availability
- Supports versioning and lifecycle rules

### RDS (Relational Database Service)
- Managed relational databases
- Supports MySQL, PostgreSQL, etc.
- Automated backups and scaling

### IAM (Identity and Access Management)
- Manages users and permissions
- Uses roles and policies
- Ensures secure access control

### VPC (Virtual Private Cloud)
- Private network in AWS
- Full control over networking setup
- Enables secure communication

### Lambda
- Serverless compute service
- Runs code without managing servers
- Triggered by events

### CloudWatch
- Monitoring and logging service
- Tracks performance metrics
- Can trigger alarms

### Auto Scaling
- Automatically adjusts resources
- Maintains performance and cost efficiency

### Elastic Load Balancer (ELB)
- Distributes incoming traffic
- Improves availability and reliability

### Route 53
- Domain Name System (DNS) service
- Routes users to applications
- Supports multiple routing policies

## Exam Key Points

- AWS follows a **pay-as-you-go pricing model**
- Supports **high availability and fault tolerance**
- Enables **scalable and flexible infrastructure**
- Security is based on **IAM roles and policies**
- Global infrastructure includes **Regions and Availability Zones**
